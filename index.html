Nostalgia Pro
Multi-Tenant WhatsApp AI Platform

Plataforma conversacional grado-producción que automatiza calificación de ventas, agendado de citas y soporte al cliente sobre WhatsApp para PyMEs en México. Inferencia on-premise por defecto; modelos en la nube reservados para tareas de visión.

Arquitectura

  • 3 agentes especializados (Sales, Scheduling, Demo) orquestados por un router de clasificación de intención.
  • Redis como memoria de corto plazo por conversación.
  • PostgreSQL como source of truth para bookings, leads y telemetría.
  • Despliegue containerizado con Docker sobre Ubuntu.

Capacidades clave

  • Consultas de disponibilidad en tiempo real, timezone-aware.
  • Protección anti-spam multi-nivel.
  • Parsing adaptativo de media (imagen, audio, documentos).
  • Telemetría de uso de tokens por agente / por cliente.
  • Escalamiento controlado a operador humano (human-handoff).

Stack

n8n LangChain PostgreSQL Redis Gemini Ollama Llama 3.2 Gemma Docker Ubuntu WhatsApp Cloud API

Por qué importa

La mayoría de PyMEs en México pierde leads por la ventana entre "vi tu mensaje" y "alguien te contesta". Nostalgia Pro cierra esa ventana sin sacrificar el control del cliente sobre sus datos — todo el modelo de lenguaje principal corre on-premise, y la base de datos es propiedad del negocio, no de un SaaS.

Market Terminal
Retro stock terminal full-stack

Resumen

Market Terminal es un terminal financiero full-stack diseñado para monitorear acciones en tiempo real, con foco en eficiencia de datos, resiliencia ante fallos de red y experiencia cross-platform. Backend FastAPI, frontend React empacado con Tauri, y un set de tooling de observabilidad para un ciclo de vida confiable de ingesta a visualización.

Objetivo

Desarrollar una plataforma que:
  • Permita exploración y análisis de datos de acciones en tiempo real sin reventar cuotas de API.
  • Mantenga usabilidad incluso en escenarios adversos (latencia, fallos parciales, límites de consumo).
  • Entregue una experiencia desktop estilo terminal retro, distinta de los dashboards típicos.

Retos & soluciones

  1. Eficiencia de llamadas a API — Cache por tipo de dato con TTL + priorización de actualizaciones críticas sobre las secundarias.
  2. Export de datos avanzado — Endpoint dedicado en FastAPI que genera un Excel con tres hojas (Basic, Financials, Technical) usando openpyxl con ajuste automático de columnas.
  3. Resiliencia del cliente — Wrappers genéricos de retry + timeout por request, auto-refresh cada 30s en la capa React.
  4. Observabilidad — HOC que inyecta métricas en cada llamada API, script diagnóstico que valida puertos, procesos activos y tamaños de cache.
  5. Validación de datos — Validadores centralizados limpian y estandarizan JSON antes de renderizar.
  6. Distribución cross-platform — Tauri empaqueta React + Vite como ejecutable ligero multiplataforma.

Arquitectura & stack

FastAPI openpyxl React + Vite Tailwind Recharts Tauri Vitest

Resultados

  • Reducción significativa de llamadas redundantes gracias al cache TTL.
  • Alta resiliencia ante fallos de red vía retries y timeouts.
  • Distribución cross-platform production-ready con Tauri.
  • Validaciones proactivas que previenen errores de UI.
