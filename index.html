Case Study.pdf

Summary

Market Terminal is a full-stack financial terminal designed to monitor stocks in real time, focusing on data efficiency, resilience to network failures, and cross-platform experience. The system uses a FastAPI backend for data management, a React frontend packaged with Tauri for native distribution, and a set of observability tools that ensure a reliable lifecycle from ingestion to visualization.

Objective
Develop a platform that:
  • Allows exploration and analysis of stock data in real time without exceeding API quotas.
  • Maintains usability even in adverse scenarios (latency, partial network outages, consumption limits).
  • Provides a retro terminal-style desktop experience, standing out from traditional dashboards.
Challenges & Solutions
  1. API Call Efficiency
    Challenge: Direct data ingestion could cause excessive quota consumption.
    Solution:
    • Implementation of data-type-based cache with time-to-live (TTL).
    • Prioritization of critical updates over secondary ones.
  2. Advanced Data Export
    Challenge: Users needed actionable reports beyond the UI.
    Solution:
    • Dedicated FastAPI endpoint generates an Excel workbook with three sheets: Basic, Financials, Technical.
    • Uses openpyxl with automatic column width adjustment.
  3. Client Resilience
    Challenge: Intermittent network outages reduced app reliability.
    Solution:
    • Generic retry + timeout wrappers for each request.
    • Auto-refresh data every 30s in the React layer.
  4. Observability
    Challenge: Diagnosing performance issues in production was difficult.
    Solution:
    • High-Order Component (HOC) to inject metrics into every API call.
    • Diagnostic script validates ports, active processes, and cache file sizes.
  5. Data Validation
    Challenge: Inconsistent JSON structures could break the UI.
    Solution:
    • Centralized validators to clean and standardize data before rendering.
  6. Cross-Platform Distribution
    Challenge: Needed a portfolio usable in both browser and desktop.
    Solution:
    • Tauri configuration packages React + Vite as a lightweight cross-platform executable.
Architecture & Stack
  • Backend: FastAPI, openpyxl, JSON cache system with metadata.
  • Frontend: React + Vite, Tailwind, Recharts.
  • Distribution: Tauri for native packaging.
  • Observability: Metrics HOC + validators.
  • Testing: Unit tests with Vitest to validate error handling and performance.
Results & Insights
  • Significant reduction in redundant calls thanks to TTL cache.
  • High resilience to network failures via retries and timeouts.
  • Production-ready cross-platform distribution with Tauri.
  • Proactive validations prevent UI errors.
  • Strengthened full-stack profile with focus on performance, observability, and holistic UX.
Lessons Learned
  • Smart cache design is key to avoiding overload of external services.
  • Packaging with Tauri offers great versatility but requires security and permission adjustments.
  • Proactive instrumentation (observability) helps identify issues before they impact users.
  • Data validators are as important as visual rendering for reliability.
